# 1. 프로젝트 설명
팀 프로젝트로 작성한 ticketon을 모듈을 분리하고 리팩토링한 리포지토리입니다.

# 2. 주요 변경점
* 멀티모듈 프로젝트 작성
    * 인증 모듈 분리
    * 대기열 모듈 분리
* 대기열 시스템 개선
    * broker 서버 확장을 고려한 설계 변경
    * dispatch 서버 유저 탈출 로직 개선
    * 유저의 대기열 중도 탈출 로직으로 인한 SSE 및 큐 정리 로직 수정
    * 단일 대기열 탈출 정보 stream으로 인한 순서 유지 문제 해결
* 인증/인가 시스템 개선
    * 인증 서버 분리
    * 인증 로직 실행 위치 일원화
    * AOP로 인가 로직 분리
* 이벤트 조회 성능 개선
    * Elasticsearch 도입해 키워드 검색 성능 개선
    * 조회 로그 저장으로 빈출 토큰 통계
* 파일 업로드 성능 개선
    * 비동기 파일 업로드로 반응속도 개선
* 결제 오류 개선
    * 결제 도중 상품의 정보가 변경되는 문제 낙관적 락으로 해결


# 3. 시도했지만 완벽하게 해결하지 못한 문제
* 결제 로직을 이벤트소스 전략으로 수정
* broker 서버가 죽었을 때 어떻게 살리는가
* common 모듈의 분리
* gateway 서버를 netty 기반의 nonblocking 서버로 교체
