## 5. 구현 계획 반영 사항
- planner가 반드시 반영할 기술 결정:
  - polling 대기열이 공식 경로이고 SSE는 deprecated이다.
  - 모든 API 응답시간은 p95 1초 미만을 목표로 한다.
  - 대기열은 동시 사용자 5,000명, 기본 polling 간격 5초, 사용자 수 기반 스케일링을 목표로 한다.
  - BC 간 일반 조회는 RabbitMQ projection으로 구현하고, 동시성이 필요한 다른 BC 조회는 내부 API로 구현한다.
  - 메시징 처리에는 outbox 패턴을 사용하고, 중복 허용/멱등 처리/동일 key 내 순서 보장을 반영한다.
  - 회원당 결제 대기 예매는 전체 1개로 제한한다.
  - 결제 제한시간은 1시간이며 만료 시 예매 자동 취소, 좌석 즉시 해제, 입장 토큰 만료를 반영한다.
  - Toss Payments만 결제/취소 provider로 취급한다.
  - Toss 결제/취소 실패는 5회 재시도 후 실패 처리한다.
  - 결제 상태 조회는 사용자 polling API로 제공한다.
  - 결제 승인 요청은 `confirm:{purchaseId}` commandId 기준으로 멱등 접수해야 하며, 동일 구매에 대해 새 outbox를 중복 생성하지 않는다.
  - 결제 승인 worker는 `messageId` inbox 중복 방지와 구매별 terminal event 검사를 함께 사용해 같은 구매의 PG 재승인, 티켓 재생성, 좌석 구매 이벤트 재발행을 막는다.
  - Toss 승인 호출에는 `confirm:{purchaseId}`를 idempotency key로 전달한다.
  - `PurchaseConfirmWorker.process`는 orchestration 수준으로 유지하고 세부 절차는 의미 단위 메서드/협력 객체로 분리한다.
  - 이벤트 수정을 우선하고, 결제 도중 이벤트 수정 감지 시 결제 트랜잭션을 롤백하고 예매를 결제 대기 상태로 되돌린다.
  - 이미지 저장은 local/external adapter를 모두 고려하고 dev profile은 local adapter를 사용한다.
  - 이미지 교체 실패 시 이전 이미지를 계속 사용하고 오류 응답을 반환한다.
  - 비밀번호만 암호화하고, 인가가 필요한 모든 API 호출의 요청 원문/응답 원문/사용자 정보/시간을 감사 로그로 남기며 1개월 보존한다.
  - 계약 테스트는 유닛 테스트로, 통합 테스트는 해피 패스 중심으로 작성하고 Toss 같은 완전 외부 의존성을 제외하고 Testcontainers를 사용한다.
  - 신규/수정 테스트 메서드명은 한글 문장형으로 작성한다.
  - 알림 기능, 프론트엔드 구현, 3D/렌더링, 클라이언트 E2E는 이번 구현 범위에서 제외한다.
- executor가 임의로 바꾸면 안 되는 결정:
  - Aggregate 내부에서 외부 시스템을 호출하지 않는다.
  - 다른 BC 내부 모델을 직접 참조하거나 수정하지 않는다.
  - 상태 변경은 aggregate root 메서드로만 수행한다.
  - 결제 금액과 좌석 합계 불일치 시 보정하지 않고 예외로 거절한다.
  - 결제 승인 멱등 기준을 `messageId`만으로 축소하지 않는다.
  - `PurchaseConfirmWorker.process`에 낮은 추상화 수준의 절차 코드를 계속 누적하지 않는다.
  - 부분 환불은 허용하지 않는다.
- 테스트/검증에 포함할 결정:
  - 좌석 중복 판매 방지 동시성
  - 회원당 결제 대기 예매 1개 제한
  - 결제 만료 시 예매 취소/좌석 해제
  - 이벤트 수정과 진행 중 결제 충돌 처리
  - Toss 승인/취소 실패 처리
  - 결제 승인 요청 중복 접수 시 outbox 중복 미생성
  - 결제 승인 worker 중복 메시지/다른 messageId 재전달 시 PG 재호출, 티켓 재생성, 좌석 구매 이벤트 재발행 방지
  - 한글 테스트 메서드명 적용
  - 이미지 업로드/교체 실패 복구
  - polling rate control과 모든 API p95 1초 미만 목표
  - outbox 메시지 중복/순서/멱등 처리
  - RabbitMQ projection과 동시성 필요 내부 API 조회
